Key things I learned
Python

Sets deduplicate automatically; True == 1 so they collapse to one element
Set operations: & (intersection), | (union), - (difference), ^ (symmetric difference)
Dict iteration pattern: for key, value in d.items():
.get(key) returns None if missing; .get(key, default) returns the default
.get() doesn't throw errors; d[key] does if key is missing

SQL

JOINs combine rows across tables using a matching key (ON clause)
INNER JOIN keeps only matched rows; LEFT/RIGHT/FULL keep unmatched ones too
Unmatched rows get filled with NULL on the missing side
Prefer LEFT JOIN over RIGHT JOIN — flip the FROM order instead
Always qualify shared column names (students.name, not just name)
Mental model: start with "what do I want ALL of" → that's FROM → LEFT JOIN the rest

Stats

Normal distribution = bell curve = Gaussian distribution
Symmetric: mean = median = mode at the peak
SD controls width: bigger SD = wider + shorter (total area always = 1)
68-95-99.7 rule: % of data within ±1, ±2, ±3 SD
Tail probabilities per side: 16% / 2.5% / 0.15%
Central Limit Theorem: many small random factors add up to a normal distribution
